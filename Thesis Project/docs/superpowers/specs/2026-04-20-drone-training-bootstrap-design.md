# Drone Training Bootstrap Design

## 背景 | Background

当前 RK3588 路线已经具备一条可运行闭环：

- `RKNN YOLOv10 + 本地视频输入 + 输出检测结果`
- `USB UVC 摄像头 + RKNN YOLOv10 + RTSP 实时推流`

但现阶段模型仍然基于通用 COCO 检测能力，无法稳定、精确地把空中无人机和鸟、飞机、风筝等干扰目标区分开。继续仅靠阈值、ROI、跟踪和运行时调优，收益已经有限。

因此下一阶段应优先建立“无人机专用模型”的最小训练闭环，并且保持与当前已跑通的 RK3588 运行链路解耦，避免为了训练工作破坏现有演示能力。

## 目标 | Goal

第一阶段目标不是做出最终高精度模型，而是尽快跑通下面这条完整闭环：

`公开无人机数据集 -> 单类 drone 训练集 -> YOLO 训练 -> pt -> onnx -> rknn -> RK3588 验证`

该阶段以“可训练、可导出、可上板验证”为优先目标。

## 范围 | Scope

本阶段包含：

- 使用公开数据集作为第一版训练数据来源
- 类别定义固定为单类 `drone`
- 建立标准 YOLO 目录与标签格式
- 训练第一版基线模型
- 导出 `onnx`
- 转换为 `rknn`
- 回到现有 RK3588 工具链中验证

本阶段不包含：

- 多类别细分训练
- 大规模数据清洗与精标
- 深度改造旧 Jetson 主工程
- UNet 迁移
- 追求最终论文级精度

## 方案选择 | Approach

评估过的方向有三种：

### 方案 A：先接入更容易转 YOLO 的公开数据集

优点：

- 最快形成训练闭环
- 风险最低
- 最适合当前“先跑通”的优先级

缺点：

- 第一版精度基础未必最好
- 后续仍需要补你自己的场景样本

### 方案 B：优先深挖当前 `Drone-detection-dataset` 的 `.mat` 标注

优点：

- 数据和任务更贴近
- 长期价值高

缺点：

- 标注格式转换成本高
- 第一阶段最容易被卡住

### 方案 C：两条数据线并行

优点：

- 长期最好

缺点：

- 第一阶段容易分散精力
- 不利于尽快出第一版结果

推荐方案：**A 为主，B 为补充。**

## 架构 | Architecture

训练相关内容将作为与现有运行链并行的新工作流存在，不直接改动当前已跑通的 `rk_yolo_video/` 与 `rk_yolo_live_rtsp/` 运行逻辑。

结构上分成四层：

1. `datasets/drone_single_class/`
   存放公开数据源、清单、转换脚本、YOLO 标签和训练集划分结果。

2. `training/` 或等价训练脚本目录
   存放训练入口、数据配置、导出脚本和训练说明。

3. 模型导出路径
   负责 `best.pt -> onnx -> rknn`。

4. RK3588 验证路径
   使用现有 `rk_yolo_video/` 和 `rk_yolo_live_rtsp/` 做回归验证。

## 数据流 | Data Flow

第一阶段的数据流如下：

1. 选择公开数据源
2. 导入到 `raw_sources/`
3. 编写转换脚本，把源标注转换为 YOLO 单类标签
4. 生成 `images/train|val|test` 与 `labels/train|val|test`
5. 生成 `dataset.yaml`
6. 训练得到 `best.pt`
7. 导出 `onnx`
8. 转成 `rknn`
9. 在 RK3588 上进行离线视频验证
10. 通过后再接入实时 RTSP 验证

## 数据源策略 | Dataset Strategy

第一阶段优先寻找“格式更容易转 YOLO”的公开无人机数据，而不是继续深挖 `.mat` 标注格式的数据。

选择标准如下：

- 有明确无人机目标标注
- 标签可被脚本化转换
- 图片或视频帧可直接提取
- 许可和下载方式尽量清晰

已有的 `Drone-detection-dataset` 继续保留在 `raw_sources/` 体系中，作为第二阶段高价值补充数据源，不在第一阶段强依赖。

## 容错与回滚 | Safety

为避免破坏现有已跑通成果，本阶段遵循以下约束：

- 训练工作全部在独立目录完成
- 不直接改现有 RK3588 运行主路径
- 新模型验证失败时，默认回退到当前稳定 `.rknn`
- 数据集下载与转换结果不直接纳入大体积 Git 跟踪

## 验收标准 | Acceptance Criteria

第一阶段完成的判定标准：

- 至少一份公开数据源被落地到数据集脚手架
- 能自动生成标准 YOLO 单类训练集
- 能跑通一次训练并得到 `best.pt`
- 能成功导出 `onnx`
- 能成功转换出 `rknn`
- 新模型能在 RK3588 上通过至少一条现有验证路径运行

## 下一步 | Next Step

实现阶段建议按以下顺序推进：

1. 选择第一份更易转换的公开无人机数据源
2. 新增导入与转换脚本
3. 生成第一版 YOLO 数据集
4. 写训练入口与导出说明
5. 训练并输出第一版模型
6. 转换为 RKNN 并回板验证
