## 高优先级审查点

### A1. INT8 任务书目标的完成边界

**可能被问：**

任务书要求“通过 INT8 量化技术，在保证精度的同时提升推理速度”，为什么你的最终稳定演示不是 full INT8？

**论文现状：**

论文已经明确说明：

- full INT8 可以加载运行，但公开视频中不能稳定出框；
- hybrid INT8 通过保护输出 Sigmoid 到最终输出层恢复检测；
- 最终稳定演示优先采用 FP RKNN；
- INT8 作为硬件优化实验和误差分析路径，而非替代 FP 主路径。

**风险等级：中。**

这不是学术硬伤，因为论文没有夸大 full INT8 成功；但任务书措辞偏强，答辩时老师可能追问“那 INT8 到底算不算完成”。

**建议答辩口径：**

本文完成了 INT8 量化链路、板端运行验证和量化误差分析。实验表明，对于小目标无人机检测，full INT8 的输出置信度对量化较敏感，出现不稳定出框；因此进一步采用 hybrid INT8 保留关键输出层精度，使检测结果恢复。最终演示选择 FP RKNN，是为了保证系统稳定性；INT8 部分作为任务书中量化优化要求的工程验证和问题分析完成。

### A2. GPIO/闭环控制未接实体外设

**可能被问：**

任务书提到 GPIO 驱动接口和报警/反制闭环，为什么没有真实继电器、蜂鸣器或云台？

**论文现状：**

论文采用了较诚实的表述：

- 当前实现软件报警 overlay；
- 记录报警事件 CSV；
- 保留 GPIO 兼容接口；
- 第六章将实体外设闭环列为不足与展望。

**风险等级：中。**

如果老师特别看重“实体硬件闭环”，这会成为追问点。但因为论文没有声称已经完成实体闭环，属于可解释边界。

**建议答辩口径：**

本课题重点最终收敛在 RK3588 端目标检测、NPU 多线程优化、RGA 与 INT8 实验验证。由于实验条件和安全性限制，未接入真实报警/反制外设。为保持系统结构完整，程序将检测结果抽象为统一报警事件输出，已经实现画面报警、事件日志和 GPIO 兼容接口；后续可将该接口连接蜂鸣器、继电器、云台或上位机平台。

### A3. RGA 已实现，但默认主路径仍需按稳定性选择

**可能被问：**

任务书要求“视频采集 -> RGA 硬件预处理 -> NPU 推理 -> 后处理”，你的 RGA 是主路径还是实验路径？

**论文现状：**

论文已经写明：

- strict RGA 配置已经板端验证；
- `RK_YOLO_REQUIRE_RGA=1` 可禁止回退 OpenCV；
- RGA 路径能够完成检测链路；
- 但不同 RGA 路径端到端收益不一致，默认演示路径应按稳定性选择。

**风险等级：低到中。**

RGA 实现证据比较充分，风险主要来自“不是默认路径”这个表达。建议答辩时强调“已实现可切换严格路径”，不要说“只是计划”。

**建议答辩口径：**

RGA 部分已经完成从 BGR/RGB 转换、缩放到 letterbox 的硬件预处理路径，并通过 `RK_YOLO_REQUIRE_RGA=1` 验证在不回退 OpenCV 的情况下完成固定视频检测。实验发现，RGA 对单阶段预处理有价值，但端到端性能还受解码、输入张量设置、后处理和推流影响。因此本文将 RGA 定位为已实现、已验证、可切换的硬件预处理路径，而最终演示是否启用取决于稳定性和场景。

## 中优先级审查点

### B1. 公开公开视频验证是否足够代表真实无人机场景

**可能被问：**

为什么没有真实无人机飞行测试？公开视频能否说明实际效果？

**论文现状：**

论文已使用公开无人机视频与实时 RTSP 进行验证，同时第六章提到真实飞行场景受测试条件限制。

**风险等级：中。**

无人机真实飞行测试受场地、安全和设备限制，解释合理。但答辩时最好不要把公开视频结果说成“真实部署完全验证”。

**建议答辩口径：**

公开视频用于提供可重复、含真实无人机目标的固定输入测试；实时 RTSP 用于验证系统实时链路和部署能力。受安全和场地限制，本文没有开展真实飞行实测，因此结论主要限定在“公开无人机视频与室内实时演示条件下的系统验证”。

**可能被问：**

**风险等级：低到中。**

**建议答辩口径：**

附录 A 选取 Edge-YOLO 是因为其关注轻量化目标检测和 RK3588 边缘部署，与本文的嵌入式目标检测系统研究高度相关。正文中无人机小目标检测背景则通过 SDD-YOLO、UAV-DETR 等资料进行补充说明。

## 对抗式结论

从最严苛角度看，当前论文主要风险集中在“任务书强要求 vs 实际工程边界”的解释，而不是格式或正文结构。论文已经通过“如实说明”的方式降低了学术风险：没有把 full INT8 说成稳定成功，没有把 GPIO 实体闭环说成已完成，也没有把 RGA 说成无条件最优。

如果继续追求更稳，最值得做的不是大改正文，而是准备答辩口径和演示材料：

1. 准备一页“任务书要求完成度对照表”；
2. 准备一页“为什么最终选择 FP RKNN 稳定演示”；
3. 准备一页“RGA / INT8 / 多 context 的实验结论与边界”；
4. 准备一句话解释 GPIO：当前是软件报警 + 事件接口，实体外设可直接接入。

## 是否建议立刻修改论文

当前不建议大改论文。若要进一步降低风险，可做两个低风险微调：

1. 将中文关键词中的“目标检测”改为“目标检测系统”，更贴合题目。
